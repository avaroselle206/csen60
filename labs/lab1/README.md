This article explains the way <a href="/en-US/docs/Glossary/HTML">HTML</a> can be used to structure a page of text by adding headings and paragraphs, emphasizing words, creating lists, and more.</p>

```
<ul>
  <li>Preferably, you should use a single <code>&lt;h1&gt;</code> per page—this is the top level heading, and all others sit below this in the hierarchy.</li>
  <li>Make sure you use the headings in the correct order in the hierarchy. Don't use <code>&lt;h3&gt;</code> elements to represent subheadings, followed by <code>&lt;h2&gt;</code> elements to represent sub-subheadings—that doesn't make sense and will lead to weird results.</li>
  <li>Of the six heading levels available, you should aim to use no more than three per page, unless you feel it is necessary. Documents with many levels (for example, a deep heading hierarchy) become unwieldy and difficult to navigate. On such occasions, it is advisable to spread the content over multiple pages if possible.</li>
</ul>
```
